vscode C++配置

Posted on 2020-03-22 Edited on 2020-03-23 In 其他 Valine:

本文主要讲述在vscode中写C++代码前进行配置碰到的问题。

之前因为vscode加载太慢，一打开vscode风扇就叫个不停，所以把所有插件都卸了。最近想写一些小型的C++代码，打开CLion又太麻烦，所以想重新在vscode中配置一下C++。现在vscode-cpptools插件已经很智能了，基本上可以开箱即用，因此本文主要记录一下我配置期间遇到的问题，至于怎么配置可以看官方文档。

问题#include errors detected. Please update your includePath. Squiggles are disabled for this translation unit (<文件名>).C/C++(1696)

好像又出现了一些其他的问题。

好像是之前改了一下settings.json中的default include path才出错的。把那个清空一下就解决了。

问题产生

这个问题是在我将各种头文件路径添加到includePath选项中，包括但不限于/usr/include、/usr/include/c++/7、/usr/include/x86_64-linux-gnu/c++/7，然后在代码中#include<iostream>就出现这种问题了。

分析

我在折腾配置文件的时候，在它的图形化配置中看到这样一句：

If on Windows with Visual Studio installed, or if a compiler is specified in the compilerPath setting, it is not necessary to list the system include paths in this list.

意思是如果指定了编译器路径，就不用将系统路径放入includePath，潜在意思是它会根据编译器自动添加includePath。

然后我就去看了一下我的gcc的includePath(命令echo | gcc -E -Wp,-v -，这条命令的意思是：-E指只进行预处理，-Wp,<option>指的是将<option>中的参数传递给预处理器，因为gcc要指定输入文件，所以这里-代表从标准输入中读取，然后echo |表示输入为空，将其传给后面的命令)：

发现貌似我需要include的那几个文件夹就/usr/include在里面。

我认为vscode-cpptools中根据编译器自动添加includePath指的是直接将上面这五个路径添加进includePath中，而gcc检查这个目录中的文件是递归检查，也就是将此目录中所有文件和文件夹都包括在内。而includePath中的内容并不是递归添加，如果要递归 添加则需要在路径末尾添加/**(If a path ends with /** the IntelliSense engine will do a recursive search for header files starting from that directory.)。
