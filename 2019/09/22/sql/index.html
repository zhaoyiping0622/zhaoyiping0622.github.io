视图

定义视图

子查询可以是任意的SELECT语句。WITH CHECK OPTION表示对视图进行UPDATE、INSERT、DELETE操作时要保证更新、插入或删除的行满足视图定义中的谓词条件。

组成视图的属性列名必须全部省略或全部指定。如果省略了视图的各个属性列名，则隐含该视图由子查询中SELECT子句目标列中的诸字段组成，但在下列三种情况下必须明确指定组成视图的所有列名：

  • 某个目标列不是单纯的属性名，而是聚集函数或列表达式。
  • 多表连接时选出了几个同名列作为视图的字段。
  • 需要在视图中为某个列启动新的更适合的名字。

例子：

  • 行列子集视图：从单个基本表导出，只去掉了基本表的某些行和某些列，但保留了主码。
  • 带虚拟列的视图：也称带表达式的视图，视图中存在由基本数据计算出的派生数据。
  • 分组视图：带有聚集函数和GROUP BY子句的查询来定义视图。

查询视图

  • 试图消解：从数据字典中取出视图的定义，把定义中的子查询和用户的查询结合起来
  • 目前多数关系数据库系统对行列子集视图丶查询均能进行正确转换。但对非行列子集视图的查询就不一定能做转换了。

更新视图

由于视图时不实际存储数据的虚表，因此对视图的更新最终要转换位对基本表的更新。

并不是所有的视图都是可更新的，一般地，行列子集视图是可更新的。

创建数据库模式的权限

  • 只有超级用户才有权创建一个新的数据库用户
  • 新创建的数据库用户由三种权限：CONNECT、RESOURCE、DBA
  • 若未指定权限，则默认为CONNECT权限，只能登陆数据库。
  • 拥有RESOURCE权限的用户能创建基本表和视图，不能创建模式，不能创建新的用户。
  • 用由DBA权限的用户是超级用户。

数据库角色

角色是权限的集合。

强制存取控制

  • 主体：用户、用户各进程
  • 客体：文件、基本表、索引、视图

主体和客体都有敏感度标记。TS(绝密)>=S(机密)>=C(可信)>=P(公开)

  • 仅当主体的许可证级别大于等于客体的密级时能读相应客体。
  • 仅当主题的许可证级别小于等于客体的密级时能写相应客体。2
