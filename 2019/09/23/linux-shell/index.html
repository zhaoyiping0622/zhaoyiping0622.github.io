readonly修饰的变量为只读变量

删除变量

unset可删除变量，变量被删除后不可再次使用。

变量类型

  • 局部变量：在脚本内部定义，仅在当前脚本实例中有效
  • 环境变量：所有程序都可以访问，必要时脚本还可修改环境变量
  • shell变量：shell程序设置的特殊变量

字符串

字符串定义可使用单引号、双引号，也可以不加引号(什么鬼？)，单引号和双引号区别：

  • 单引号中不可使用变量，任何字符都原样输出
  • 单引号中不能出现单独的单引号，转义之后也不行，但可成对出现作为字符串拼接使用
  • 双引号之中可以有变量
  • 双引号中可以有转义字符

定义数组

使用小括号标志范围，值用空格隔开

也可以单独定义数组各个分量，下标不用连续

读取数组

${数组名[下标]}

获取数组长度方法与获取字符串长度相同

传递参数

在运行脚本时可以传入参数

在脚本中$n表示第n个参数，$0表示文件名。

test.sh:

运行参数：

./test.sh 1 2 3 4

输出：

./test.sh
1
2
3
4

参数 用途
$# 参数个数
$* 以字符形式显示所有参数
$$ 脚本运行的当前进程号
$! 后台运行的最后一个进程的ID
$@ 与$*类似 但是使用时需加引号
$- 显示Shell使用的当前选项
$? 最后命令退出状态

$*与$@区别：

  • $@ 等价于三个参数，$* 等价于一个由三个参数拼接的一个参数

关系运算符

  • -eq检查两个数是否相等(equal)
  • -ne检查两个数是否不相等(not equal)
  • -gt检查左边的数是否大于右边的数(greater than)
  • -lt检查左边的数是否小于右边的数(less than)
  • -ge检查左边的数是否大于等于右边的数(greater equal)
  • -le检查左边的数是否小于等于右边的数(less equal)

布尔运算符

  • !非运算
  • -o或运算(or)
  • -a与运算(and)

逻辑运算符

  • &&
  • ||

字符串运算符

  • =检查两个字符串是否相等
  • !=检查两个字符串是否不相等
  • -z检查字符串长度是否为0(zero)
  • -n检查字符串长度是否为0(not zero)
  • $检查字符串是否为空，不为空返回true

文件测试运算符

  • -b file检查文件是否是块设备文件(block)
  • -c file检查文件是否是字符设备文件(char)
  • -d file检查文件是否是目录(directory)
  • -f file检查文件是否是普通文件(file)
  • -g file检查文件是否设置了SGID位
  • -k file检查文件是否设置了粘着位
  • -p file检查文件名是否是有名管道(pipe)
  • -u file检查文件是否设置了SUID位
  • -r file检查文件是否可读(read)
  • -w file检查文件是否可写(write)
  • -x file检查文件是否可执行
  • -s file检查文件是否为空
  • -e file检查文件(包括目录)是否存在(exist)
  • -L file检查文件是否存在并且是个符号链接
